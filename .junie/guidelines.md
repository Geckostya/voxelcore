# Руководство проекта для Junie (.junie/guidelines.md)

Краткий обзор и правила работы ИИ‑ассистента (Junie) в репозитории VoxelCore.

— Этот проект: C++17, Windows, CMake + vcpkg. Основной исполняемый модуль: VoxelEngine. Подробнее о запуске и первой сцене см. SETUP.md в корне репозитория.

Проект: VoxelEngine (VoxelCore)
- Язык/стандарт: C++17
- ОС/IDE: Windows 10+, Rider/Visual Studio
- Сборка: CMake, vcpkg
- Решение: VoxelEngine.sln

Структура (упрощённо)
- res — ресурсы движка (ядро)
- src — исходники ядра/движка
- test — модульные/интеграционные тесты
- build, x64 — артефакты сборки (генерируются CMake/MSBuild)
- user — пользовательские данные (создаётся автоматически устройством user:)
- project — папка проекта (устройство project:, может совпадать с res при отсутствии явного указания)
- content — пакеты контента (внутри соответствующих корней: world:, user:, project:, res:)

Монтаж путей (виртуальные устройства)
- res: → папка ресурсов
- user: → папка пользовательских данных (создаётся при первом запуске)
- project: → папка проекта
- world: → текущий мир (создаётся при создании мира)

Как собирать и запускать
- Подробно описано в SETUP.md. Кратко: откройте VoxelEngine.sln, соберите Debug x64/RelWithDebInfo и запустите. Если project:project.toml отсутствует, будет создан минимальный файл по умолчанию.

Тесты
- Тестовые проекты находятся в каталоге test (есть соответствующие цели CMake/VS: RUN_TESTS, VoxelEngineTest и т.д.).
- Для Junie: предпочтительно запускать тесты через инструмент run_test (если доступен) или через RUN_TESTS в решении. Примеры:
  - run_test fullSolution — запустить все доступные тесты в решении.
  - Либо собрать цель RUN_TESTS в Visual Studio/Rider.

Сборка перед отправкой
- Если изменения касаются C++ кода или CMake-конфигурации, необходимо убедиться, что проект собирается (Debug x64 достаточно). Для правок документации/ассетов сборка не обязательна.

Код‑стайл и форматирование
- C++17, заголовки .hpp, реализации .cpp.
- Отступы: 4 пробела (как в текущем коде).
- Имена и стиль — следовать текущим практикам проекта.
- ВАЖНО: комментарии в коде должны быть исключительно на английском языке. Русский язык допустим в документации (Markdown), но не в исходниках. 

Логи и отладка
- Основной лог запуска: latest.log (в каталоге сборки или рядом с исполняемым файлом). Дополнительные логи по подсистемам — через debug::Logger.

Принципы изменений, вносимых Junie
- Делайте минимально необходимые правки для выполнения задачи.
- Сохраняйте обратную совместимость и избегайте ломающих изменений без крайней необходимости.
- Сопровождайте изменения кратким описанием (commit/patch message) и при необходимости ссылками на соответствующие части кода.
